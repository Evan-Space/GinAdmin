# Skill: 新增或改造 Ant Design Form

当新增或改造表单、筛选表单、编辑表单时，使用本 skill。

## 适用场景

- 登录表单。
- 查询筛选表单。
- 新增/编辑用户。
- 新增/编辑角色和权限。
- 系统设置表单。
- AI 生成表单建议的预览和应用。

## 表单类型

### 查询表单

- 通常使用 `layout="inline"`。
- 字段数量较多时可以拆成展开/收起。
- 提供查询和重置按钮。
- 查询参数应由 hook 或 page state 管理。

### 编辑表单

- 通常放在 `Drawer` 或 `Modal` 中。
- 使用 `Form.useForm()`。
- 提交中禁用确认按钮。
- 提交成功后关闭弹层并刷新列表。
- 提交失败时展示明确错误。

## 字段规则

- 必填字段使用 `rules`。
- option value 要有明确类型。
- 密码、token、secret 等敏感字段不要进入 AI context。
- 日期范围字段要明确转换为后端需要的格式。
- 不要让用户输入被静默清空。

## 提交规则

推荐提交流程：

1. 用户填写表单。
2. `form.validateFields()` 校验。
3. 禁用提交按钮。
4. 调用 API。
5. 成功后提示并刷新数据。
6. 失败后提示错误并保留用户输入。

## AI 增强

适合 Form 的 AI 能力：

- 为非敏感字段生成填写建议。
- 根据描述生成权限草案。
- 解释某个字段的含义。
- 检查表单配置是否存在明显冲突。

AI 生成的字段值必须以 preview 展示，用户确认后才写入表单。

## 常见错误

- 没有校验 rules。
- pending 时可以重复提交。
- 请求失败后表单直接关闭。
- select option value 没有类型。
- AI 建议直接覆盖用户输入。

## 验收标准

- 字段类型明确。
- required、长度、格式等关键校验存在。
- pending、success、error 状态完整。
- 用户输入不会被静默丢失。
- AI 生成值只作为建议，确认后才应用。
